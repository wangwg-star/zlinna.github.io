测试角度学习 cache2go

Aug 20th 2019

cache2go 介绍

“Concurrency-safe golang caching library with expiration capabilities.”。作者如是介绍该项目，翻译后的意思为：存在心跳机制且并发安全的go语言缓存库。

测试角度学习 cache2go

我们知道心跳机制、并发安全是 cache2go 的两大特点，那么肯定是测试的重点了。

首先查看 cache_test.go ，可以获得以下信息：【Cachetable 简称 table，cacheitem 简称 item】
| 测试点 | 具体因子 | 描述 |
| - | - | - |
| TestCache | Cache, table.Add, table.Value, item.AccessCount, item.LifeSpan, item.AccessedOn、item.CreateOn | 创建 table ，添加 item ，获取 item 基本信息 |
| TestCacheExpire | Cache, table.Add, table.Value | 心跳机制：超出 lifeSpan 存活时间后自动消除 item |
| TestExists | Cache, table.Add, table.Exists | table 表中是否存在 item |
| TestNotFoundAdd | Cache, table.Add, table.NotFoundAdd | table 表中是否存在 item ，若不存在则添加 |
| TestNotFoundAddConcurrency | Cache, table.NotFoundAdd, table.Foreach | 并发安全： NotFoundAdd |
| TestCacheKeepAlive | Cache, table.Add, item.KeepAlive, table.Exists | 心跳机制： item 重新计时 |
| TestDelete | Cache, table.Add, table.Value, table.Delete | 从 table 表中删除 item |
| TestFlush | Cache, table.Add, table.Flush, table.Value | 清空 table 表所有 item |
| TestCount | Cache, table.Add, table.Value, table.Count | table 表中 item 数量 |
| TestDataLoader | Cache, table.SetDataLoader, NewCacheItem, table.Value, table.Exists, item.Data | 访问不存在的 item 时会触发 loadData 回调函数 |
| TestAccessCount | Cache, table.Add, table.Value, table.MostAccessed | 返回访问频率高的 count 条 item |
| TestCallbacks | Cache, table.SetAddItemCallback, table.SetAboutToDeleteItemCallback, item.SetAboutToExpireCallback, table.Add | 增加、删除或超时移除情况下触发回调 |
| TestCallbackQueue | Cache, item.Key, table.Add, table.AddAddedItemCallback, table.AddAboutToDeleteItemCallback, item.AddAboutToExpireCallback, table.RemoveAddedItemCallbacks, table.RemoveAboutToDeleteItemCallback, item.SetAboutToExpireCallback, item.RemoveAboutToExpireCallback | 设置、增加、清除回调函数正常 |
| TestLogger | Cache, table.SetLogger, table.Add | 设置 logger 正常 |

基准测试 具体因子 描述
BenchmarkNotFoundAdd Cache, table.NotFoundAdd 基准测试： NotFoundAdd
